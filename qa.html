Namagame Creator Q&A

最終更新日: 2026年2月24日

Q. Namagame Creator を Windows で開くと「Windows によって PC が保護されました」と表示されます。

現在は試作版のため、コード署名（デジタル証明書）が未設定でこの警告が表示される場合があります。 その場合は、ダイアログの「詳細情報」をクリックし、右下に表示される「実行」を押すことで起動できます。

Q. Namagame Creator の Mac 版がないのはなぜですか？

現在、Mac 版のビルドと配布の整備が完了していないためです。対応が完了し次第、提供予定です。

Q. 「アプリケーションへのネットワーク受信接続を許可しますか？」というダイアログが表示されます。許可して大丈夫ですか？

はい、許可してください。許可しない場合、アプリの機能が一部利用できず、エラーになることがあります。

Q. ゲームは生成されましたが、エラーが出て動きません。

CODE タブを開き、コード下部のエラーメッセージをコピーして、 「〜というエラーが出ていてゲームが動かないので修正してください」のように入力し、ゲーム修正 ボタンを押してください。 コード下部にエラーメッセージが表示されない場合は、デバッグ画面を開いて内容を確認してください。

Q. 画像や音声をゲームに入れるにはどうすればいいですか？

メッセージ内で画像・音声のファイルパスを指定してください。以下のように指示するとスムーズです。

例: 以下で指定した画像素材と音声素材をコピーして、ゲームプロジェクトのディレクトリに配置してください。 画像は image ディレクトリ、音声は audio ディレクトリに配置し、ゲームから利用してください。

  • 画像素材: サンプル画像 C:\Users\hoge\workspace\game\image\sample.png
  • 音声素材: BGM C:\Users\hoge\workspace\game\audio\bgm.mp3
  • 音声素材: SE C:\Users\hoge\workspace\game\audio\se.mp3

Q. 音声が鳴りません。

まず game.json に対象の音声ファイルが登録されているか確認してください。 登録されていない場合は、メッセージで明示的に「音声ファイルを m4a / ogg 形式に変換して登録する」と指示するのがおすすめです。 登録済みでも再生できない場合は、エラー発生時と同じ手順でエラーメッセージを確認し、修正を依頼してください。

Q. 完成前ですが、途中まで作った内容を保存しておきたいです。

「ニコ生ゲームをダウンロード」の右隣にある「▽」ボタンを押し、「プロジェクト」を選択してください。 生成したゲームを、指定した場所に zip 形式で保存できます。

Q. 既存のニコ生ゲームの改造はできますか？

「既存のプロジェクトの読み込み」から可能です。 ただし本アプリがサポートしている言語は JavaScript のため、TypeScript で開発されたプロジェクトを読み込む場合は、ビルド後の JavaScript を対象にしてください。

Q. おすすめの設計モデルと実装モデルは何ですか？

一例として、設計モデルは gpt-5-mini、実装モデルは gpt-5.2-codex をおすすめします。

Q. OpenAI の APIキーは有料ですか？

はい、有料です。初回登録時に無料クレジットが付与される場合もありますが、継続利用にはクレジット購入が必要です。

Q. 1つのゲーム生成にかかる料金はどれくらいですか？

ゲームの内容や規模によって変わるため一概には言えませんが、目安として、1分程度のランキング形式のニコ生ゲームを生成・修正する場合、約 2 USD 程度になることがあります。

Q. アプリに不具合がある場合、どこに連絡すればよいですか？

以下の GitHub Issues へ、不具合内容を記載してください。状況により対応できない場合があります。

https://github.com/dera-/namagame-creator-app-win/issues

Q. アプリ本体ではなく、内部で使っている MCP サーバーだけを使いたいです。

akashic-mcp をローカルで起動し、MCP サーバー対応の任意の LLM エージェントから利用できます。 詳細は以下を参照してください。

https://github.com/dera-/akashic-mcp
